# Tesla B/W Design System – CSS 변수 레퍼런스

> 본 문서는 `src/styles/design-system.css` 에 정의된 **전역 CSS 변수**의 의미와 사용 지침을 상세하게 설명한다. 라이트 모드를 기준으로 설명하며, 다크 모드는 동일 변수명을 재정의(overwrite)하여 동작한다.

---

## 1. 기본 배경 · 텍스트 계열

| 변수 | 라이트 값 | 다크 값 | 역할 | 주 사용 위치 |
|------|-----------|---------|------|--------------|
| `--background` | `0 0% 100%` | `0 0% 8%` | 문서 전체 배경색 | `body`, `html` |
| `--foreground` | `0 0% 5%` | `0 0% 95%` | 기본 텍스트·아이콘 | 모든 텍스트 요소 |
| `--card` | `0 0% 98%` | `0 0% 12%` | 카드·패널 배경 | `.neu-flat`, `.card` |
| `--card-foreground` | `0 0% 10%` | `0 0% 90%` | 카드 내부 텍스트 | 카드 제목·내용 |
| `--muted` | `0 0% 94%` | `0 0% 18%` | 비활성·보조 배경 | placeholder, disabled |
| `--muted-foreground` | `0 0% 40%` | `0 0% 65%` | 비활성 텍스트 | placeholder text |
| `--border` | `0 0% 88%` | `0 0% 25%` | 테두리·구분선 | `border-color`, `hr` |

### 사용 팁
- 배경색을 지정할 때는 **항상** `hsl(var(--변수명))` 형태를 사용한다. 투명도 필요 시 `/ alpha` 슬래시 표기.
- 예) `background: hsl(var(--muted) / 0.7);`

## 2. 강조 색 · 브랜드 계열

| 변수 | 라이트 값 | 다크 값 | 의미 | 비고 |
|------|-----------|---------|------|------|
| `--primary` | `0 0% 20%` | `0 0% 85%` | 핵심(브랜드) 색 | 버튼, 링크, 아이콘 활성 |
| `--primary-rgb` | `51, 51, 51` | `217, 217, 217` | `rgba()` 전용 RGB 배열 | `filter: drop-shadow()` 등 |
| `--primary-foreground` | `0 0% 98%` | `0 0% 8%` | 프라이머리 배경 위 텍스트 | 반전 대비 확보 |
| `--secondary` | `0 0% 30%` | `0 0% 70%` | 보조 강조색 | 서브 버튼, 태그 |
| `--secondary-foreground` | `0 0% 98%` | `0 0% 8%` | │ |
| `--accent` | `0 0% 40%` | `0 0% 60%` | 하이라이트 | hover 배경, 마커 |
| `--accent-foreground` | `0 0% 98%` | `0 0% 8%` | │ |

> **주의** : 브랜드 컬러를 바꾸고 싶다면 `--primary` 와 `--primary-rgb` 두 값을 **동시에** 수정해야 일관성이 유지된다.

## 4. 뉴모피즘 전용 변수

| 변수 | 타입 | 라이트 | 다크 | 설명 |
|-------|------|--------|------|------|
| `--neu-light` | `R,G,B,A` | `255, 255, 255, 0.95` | `255, 255, 255, 0.05` | 입체 효과의 **하이라이트** 색 |
| `--neu-dark` | `R,G,B,A` | `0, 0, 0, 0.08` | `0, 0, 0, 0.35` | 입체 효과의 **그림자** 색 |

| `--neu-blur` | `px` | `6px` | `5px` | 그림자 blur 반경 |
| `--neu-inset-light` | `R,G,B,A` | `0, 0, 0, 0.15` | `0, 0, 0, 0.6` | **음각(inset)** 내부 그림자 |

### 빠른 예제
```css

  @apply neu-raised;
  color: hsl(var(--primary-foreground));
```

## 6. 네이밍 규칙 정리

1. **역할 + 속성** : `primary-foreground`, `destructive-foreground` 처럼 배경·텍스트 쌍으로 유지.
2. **RGB 리스트** : `-rgb` 접미사는 `rgba(var(--primary-rgb), 0.2)` 같이 사용.
3. **뉴모피즘** : 접두사 `neu-` 는 그림자·하이라이트·길이를 분리한다.
4. 색상 값은 **항상 HSL 세 트리플** 또는 **RGB 리스트**로 저장한다. 정수형 `#hex` 사용 금지.

## 8. 변수별 상세 가이드

### --success

| 항목 | 설명 |
|------|------|
| **역할** | 성공, 완료, 확인 등의 긍정 상태를 나타내는 메인 색상 |
| **라이트 값** | `142 71% 45%`  → HSL (녹색 계열) |
| **다크 값** | `142 71% 55%`  → 대비 확보를 위해 채도·명도 ↑ |
| **쌍으로 쓰는 변수** | `--success-foreground` (`0 0% 98%` / `0 0% 8%`) |
| **주요 사용처** | • 토스트/알림 <br/>• 입력 검증 성공 메시지 <br/>• 그래프·차트의 긍정 데이터 <br/>• 배지/라벨(성공) <br/>• 프로그레스바·스텝 완료 표시 |
| **권장 조합** | `background: hsl(var(--success));` + `color: hsl(var(--success-foreground));` |
| **접근성** | WCAG 2.1 대비: 텍스트 대비 4.5:1 이상 확보. <br/> `--success` 배경 위 텍스트는 반드시 `--success-foreground` 사용. |
| **주의사항** | 1) 브랜드 컬러보다 우선순위 낮다. <br/>2) 상태 표현 이외 용도로 남용 금지. |

#### 사용 예시

```tsx
// ✅ 성공 배지
<span className="px-2 py-1 rounded-md text-xs font-medium" 
      style={{
        background: 'hsl(var(--success))',
        color: 'hsl(var(--success-foreground))'
      }}>
  성공
</span>

// ✅ 성공 토스트 (Radix Sonner)
<Toaster>
  <Toast action="success" style={{
    background: 'hsl(var(--success))',
    color: 'hsl(var(--success-foreground))'
  }}>
    저장이 완료되었습니다.
  </Toast>
</Toaster>
```
